# AI简历助手

一个适用于 Codex、面向技术岗位的简历写作教练技能。

它不会从零代写简历，也不会沿用旧简历的框架做表面润色。它先把旧简历拆解为独立事实和证据，再分析岗位方向、内容强弱与结构问题，引导用户自行设计新框架、写出新文本。全部文字经用户确认后，可以继续整理版式并生成最终的可携带文档格式（PDF）文件。

## 核心原则

- 不从空白页面生成完整简历或项目经历。
- 旧简历只有事实来源权，不自动继承其章节、顺序、项目分组和措辞。
- 助手不提供新框架，由用户根据证据分析自行设计。
- 每次只处理一个小节或少量相关条目。
- 局部修改只能针对用户新写的文本，并使用已确认事实。
- 不虚构指标、职责、技术栈、业务背景或因果关系。
- 最终内容必须可追溯为用户原写或用户确认过的局部修改。
- 检查重复句式、平均用力、宏大空话和关键词拼接，避免流水线式人工智能生成文风。
- 支持岗位定位、项目叙事、可信度审查、岗位描述匹配和最终 PDF 制作。

## 工作流程

1. 读取旧简历，将其中的日期、经历、职责、技术选择、指标和成果拆成独立证据。
2. 分析旧框架存在的问题，但不继承原有章节顺序、项目分组和篇幅分配。
3. 结合目标岗位，整理证据强弱、岗位覆盖和仍需补充的信息。
4. 通过少量问题，引导用户自行设计新简历框架。
5. 由用户逐段写出新文本，助手只提供事实核验、问题诊断和局部修改建议。
6. 检查时间线、职责边界、指标可信度、岗位指向和模板化文风。
7. 用户确认全部内容后，统一格式并按需生成 PDF。

## 安装

将 `skills/write-impactful-tech-resume` 目录复制到：

```text
$CODEX_HOME/skills/write-impactful-tech-resume
```

未设置 `CODEX_HOME` 时，通常使用：

```text
~/.codex/skills/write-impactful-tech-resume
```

## 使用示例

```text
使用 $write-impactful-tech-resume 引导我逐段完成技术简历。不要从零生成内容，先从最重要的项目开始提问。
```
